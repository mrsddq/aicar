# Notebook Review Checklist

Before treating a notebook as complete, confirm:

- The notebook runs from a clean kernel.
- Imports are grouped at the top.
- Data paths are relative, not machine-specific.
- Markdown explains the goal before code begins.
- Every model has a metric.
- Visualizations have titles and axis labels.
- Final markdown summarizes what worked and what failed.
- Large outputs are cleared before committing.
